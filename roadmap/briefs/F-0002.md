## Feature
Add Prisma ORM with an embedded SQLite database: core models `User`, `Project`, `Question`, `Answer`, `Source`, `Chunk` with relations and status fields, plus a `seed` npm script that pushes the schema and populates realistic fixtures. The gate's E2E seed shim (`scripts/seed.ts`) delegates to `npm run seed`, so that script name is a contract. Spec: `roadmap/ROADMAP.md#now` (skeleton epic). Nearly every later feature (workspace, RAG, clusters, exports) builds on this data layer and its seed corpus.

## Hard constraint ‚Äî read first
**The `seed` npm script must NOT invoke `scripts/seed.ts` in any way** (no `ts-node scripts/seed.ts`, no `npm run` chain that reaches it). The gate shim `scripts/seed.ts` delegates *to* `npm run seed`; pointing back at it creates circular delegation. The shim has a `SEED_SHIM_ACTIVE` sentinel that fails such loops, but your script must simply run the real seeding logic directly. `scripts/**` is forbidden surface for you anyway ‚Äî never touch it.

## Stack facts (live-verified 2026-06-11 ‚Äî do NOT fall back to older Prisma idioms from memory)
- Current stable: **prisma 7.8.0** and **@prisma/client 7.8.0** (released 2026-04-22). Install both pinned to the same version (`prisma` as devDependency, `@prisma/client` as dependency).
- **Prisma 7 differs sharply from v6.** All of the following v6 idioms are REMOVED in v7 ‚Äî using them will fail:
  - `generator client { provider = "prisma-client-js" }` ‚Üí now `provider = "prisma-client"`, which emits plain TypeScript and **requires** an `output` path (no longer generates into node_modules).
  - `new PrismaClient()` with no adapter ‚Üí SQLite now **requires a driver adapter**: `@prisma/adapter-better-sqlite3` (class name `PrismaBetterSqlite3` in v7), which depends on native `better-sqlite3` ^12.x. Construction: `new PrismaClient({ adapter: new PrismaBetterSqlite3({ url: "file:./prisma/dev.db" }) })`.
  - `url` in the schema `datasource` block ‚Üí moved to **`prisma.config.ts` at the repo root** (required for CLI commands like `db push`). Use `defineConfig` from `prisma/config`.
  - `package.json#prisma` (both `schema` and `seed` keys) ‚Üí REMOVED. Seed hook lives in `prisma.config.ts` under `migrations: { seed: "tsx prisma/seed.ts" }`.
  - The CLI no longer auto-loads `.env` and `@prisma/client` no longer auto-generates on postinstall.
- `prisma db push` is the documented migration-less sync for a dev SQLite DB. In v7 it does **not** trigger generators ‚Äî `prisma generate` must run separately.
- Official docs run TS seed files with **tsx** (not ts-node). Add `tsx` as a devDependency. (The repo's engine scripts keep using ts-node ‚Äî leave them alone.)
- Known TS6 issues (#29549 declaration-emit, #29449 loose `data:` typing) do NOT affect this repo's `tsc --noEmit` gate. No action needed.

### 2. Prisma schema ‚Äî `prisma/schema.prisma`
Datasource `provider = "sqlite"` (no url here ‚Äî v7 puts it in prisma.config.ts). Generator:
```prisma
generator client {
  provider = "prisma-client"
  output   = "./generated"
}
```
(i.e. generated client lands in `prisma/generated/` ‚Äî deliberately OUTSIDE `src/` so `biome lint scripts src` never sees machine-generated code. Import it from `src/lib/prisma.ts` via a relative path.)

Models (lean ‚Äî later features extend via `db push`; SQLite provider does not support Prisma enums, so status fields are `String` with documented values):
- `User`: id (cuid), name, email `@unique`, createdAt.
- `Project`: id, name, createdAt; questions relation.
- `Question`: id, projectId FK (`onDelete: Cascade`), text, category `String?`, section `String?`, order `Int` (original row order), status `String @default("unanswered")` ‚Äî allowed values documented in a comment: `unanswered | drafted | approved`; answer relation.
- `Answer`: id, questionId FK `@unique` (one answer per question, `onDelete: Cascade`), text, createdAt, updatedAt.
- `Source`: id, name, type (`txt|pdf|docx|seed`), createdAt; chunks relation (`onDelete: Cascade` on chunk side).
- `Chunk`: id, sourceId FK, position `Int`, text; `@@unique([sourceId, position])`.

### 4. Seed logic ‚Äî `src/lib/seed.ts` (function) + `prisma/seed.ts` (entry)
